# Code of Conduct

## Our standard

BigDiff should stay practical, respectful, and safe for contributors and users.

Examples of expected behavior:

- be respectful and specific in technical discussions
- assume good intent while reviewing code or bug reports
- share reproducible examples when possible
- protect private, clinical, or otherwise sensitive data

Examples of unacceptable behavior:

- harassment, personal attacks, or discriminatory language
- posting private data, production files, or security-sensitive details without consent
- deliberately hostile review behavior or repeated bad-faith disruption

## Scope

This applies to issues, pull requests, code review, and other project spaces.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainer privately when possible. If the report also involves a security concern, use the path described in `SECURITY.md`.

## Enforcement

The maintainer may edit, hide, lock, or remove content that violates this code of conduct and may restrict participation when needed.
