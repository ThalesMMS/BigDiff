# Security Policy

## Supported versions

This is an early-stage project. The latest commit on the default branch is the only supported version for security fixes.

## Reporting a vulnerability

Please do not open a public issue with exploit details.

Because this repository is private, contributors are expected to use a private channel they already have with the maintainer. Include:

- affected implementation (`Rust` or `Python`)
- impact and attack scenario
- reproduction steps or proof of concept
- whether sensitive files or data are involved
- any suggested mitigation

Private reporting endpoint: use GitHub Security Advisories at https://github.com/ThalesMMS/BigDiff-dev/security/advisories/new for a non-public report.

If you do not already have a private contact path, open a minimal issue that says you need a private security handoff, without including exploit details.

## Response expectations

The maintainer will try to acknowledge reports within 7 days and follow up with next steps when the issue is confirmed.
